LCS (Lecture Capture system)

OverView

This project aims to create an automatic audio-video recording system for classroom lectures. Its primary goal is to enable students who have missed lectures to access them conveniently online for exam preparation or reference at any time and from anywhere.

Hardwares Connectivity

The Raspberry Pi, IP camera, and audio injector are all connected via LAN cables and set up in classrooms. All classroom devices are connected to the same network, and these devices are mapped to a file server based on storage, which is further mounted in one application server.

Modules

  • Server Time: This module checks the server time and sets the correct timing on Raspberry Pi devices.
  • Mount: It verifies the connectivity between Raspberry Pi and the file server.
  • Lecture Scheduling: Professors update the scheduled date, timing, duration, subject, department, and other details through a web portal.
  • Capture: Based on lecture timing, this module initiates and stops recording using IP cameras and stores the recordings on a Pendrive.
  • Move: It transfers the recorded videos from the Pendrive to a specific file server path.

Virtual Scribe

Overview

Virtual Scribe is the web and windows application to record the patient’s clinical information while the physician is treating the patient. The authorized Medical scribe will listen to the audio file and document the same in client’s EHR System from the recorded information.

Modules

Common Module
  • Login
  • OTP Verification
  • New User Update Password
  • Forgot Password
  • Reset Password
Vee Manager,Super Admin and Admin

  • Vee manager, Super admin and Admin this modules are have similar functionalities. User can able to do CRUD operations based on role hierarchy order.

Physician
  • This module is used to record the patient clinical information.
  • The information will be saved as an audio file in their local environment.
  • The physician can able to stop, pause and restart the audio recording. Finally, they can upload the recorded audio file to Amazon AWS S3 buckets.
  • Once the file is uploaded, application will verify the file size in both environment (local and AWS) and show the confirmation to them.
  • Then, the recorded file will be deleted from their local environment.
